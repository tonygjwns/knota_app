# KNOTA_app 문서

작업이 누적되면서 기억해 두면 좋을 결정, 흐름, 화면 명세를 모은 폴더.

## 어디부터 봐야 하나

| 파일 | 언제 보는지 |
|---|---|
| [architecture.md](architecture.md) | 시스템 한눈에 — 데이터 모델, 권한, 핵심 흐름. 작업 재개 시 컨텍스트 회복용 |
| [decisions.md](decisions.md) | 시간순 결정 기록 (ADR). "이게 왜 이렇게 됐지?" 추적용 |
| [specs-v2/](specs-v2/) | 화면/도메인별 상세 명세 — 사용자가 작성한 큰 그림 문서들 |
| [specs/](specs/) | 옛 명세 — 참고용. specs-v2 가 더 최신 |

## 작업 흐름과의 관계

- base44 에 작업 지시 시 — 관련 specs-v2 또는 architecture.md 의 섹션을 참조 표시
- 큰 결정 발생 시 — decisions.md 에 한 줄 추가
- 새 화면/흐름 — specs-v2 에 새 파일 또는 기존 파일 갱신

## 파일 작성 규칙

- 결정은 *시간순* 으로 누적 (덮어쓰기 X). 옛 결정이 번복되면 새 항목으로 추가하고 옛 항목은 그대로 둠 (이력 보존)
- 화면 명세는 *현재 동작* 만 — 코드와 어긋나면 갱신
- 한국어 + 마크다운. 코드 인용은 ` ``` ` 블록
